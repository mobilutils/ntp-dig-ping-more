# Walkthrough - Android 15 (API 35) Work Profile Setup & Run
This developer guide provides step-by-step instructions on how to set up an Android 15 Work Profile (Managed Profile) on an emulator using command-line tools (`adb`), install the NTP DIG PING MORE application inside the work profile, and run/debug it.

This is highly useful for verifying MDM (Mobile Device Management) scenarios, Managed Configurations, and profile separation behaviors in a BYOD (Bring Your Own Device) environment.

---

## Prerequisites

Before starting, ensure that:
1. You have the **Android SDK** installed and `adb` is available in your shell's `PATH`.
2. An Android Emulator (preferably running **Android 15**) is booted and connected to ADB.
3. You have the Test DPC APK (`testdpc.apk`) and the project's debug APK (`app-debug.apk`) available locally.

---

## Step 1: Launch and Reset the Emulator

To ensure a clean environment without any conflicting user accounts, it is recommended to start the emulator with wiped user data:

```bash
emulator -avd AVD_NAME -wipe-data
```

Ensure the emulator is fully booted by verifying:

```bash
adb devices
# Output should show: emulator-5554	device

adb shell getprop sys.boot_completed
# Output should be: 1
```

---

## Step 2: Create a Work Profile via CLI

To create a managed work profile associated with the primary user (User `0`), execute the following command:

```bash
adb shell pm create-user --profileOf 0 --user-type android.os.usertype.profile.MANAGED "Work Profile"
```

### Expected Output
```text
Success: created user id 10
```

> [!NOTE]
> The created user ID (typically `10` or higher) will be referred to as `USER_ID` in all subsequent commands.

Start the newly created profile user:

```bash
adb shell am start-user USER_ID
```

---

## Step 3: Setup Test DPC as the Profile Owner

Test DPC is an enterprise utility used to simulate device policy constraints. 

1. Install **Test DPC** to the managed profile user:
   ```bash
   adb install --user USER_ID testdpc.apk
   ```

2. Register Test DPC as the official **Profile Owner** of the work profile:
   ```bash
   adb shell dpm set-profile-owner --user USER_ID com.afwsamples.testdpc/.DeviceAdminReceiver
   ```

### Expected Output
```text
Success: Active admin and profile owner set to com.afwsamples.testdpc/.DeviceAdminReceiver for user 10
```

---

## Step 4: Install the Application in the Work Profile

Install the project's debug APK inside the work profile using the `--user` flag:

```bash
adb install --user USER_ID ./app/build/outputs/apk/debug/app-debug.apk
```

### Expected Output
```text
Performing Streamed Install
Success
```

---

## Step 5: Launch the Application in the Work Profile

Launch the NTP DIG PING MORE application launcher activity targeting the work profile user:

```bash
adb shell am start --user USER_ID -n io.github.mobilutils.ntp_dig_ping_more/.MainActivity
```

### Expected Output
```text
Starting: Intent ( cmp=io.github.mobilutils.ntp_dig_ping_more/.MainActivity )
```

In the emulator screen, you should now see the application open with a small **briefcase badge** on the bottom right of its launcher icon and task switcher bar, confirming that it is successfully running inside the secure **Work Profile** partition!

---

## Useful Verification Commands

### List Active User Profiles
To list all active user profiles on the device:
```bash
adb shell pm list users
```
*Expected output shows User 0 (Primary) and User 10 (Work Profile).*

### List Installed Packages for the Work Profile
To list packages specifically installed inside the work profile:
```bash
adb shell pm list packages --user USER_ID | grep mobilutils
```

### Remove/Clean up the Work Profile
To delete the work profile and completely reset:
```bash
adb shell pm remove-user USER_ID
```
