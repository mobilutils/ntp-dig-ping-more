## Paramètres

<div style={{ display: 'flex', gap: '24px', alignItems: 'flex-start', marginBottom: '32px' }}>
      <div style={{ flex: 1 }}>
         - **Délai d'attente opérationnel** — délai global (1–60 s) appliqué à tous les outils réseau
           - Modifications sauvegardées immédiatement sur entrée valide ; annulation au focus-perdu si invalide
         - **Configuration Proxy PAC** — surcharge proxy au niveau application (indépendante du proxy système)
           - Activer/désactiver le routage proxy
           - Champ URL PAC avec validation de format URL (http/https uniquement)
           - Scripts PAC évalués avec un moteur JavaScript léger (QuickJS)
           - Résultats PAC mis en cache avec TTL de 5 minutes pour les performances
           - Supporte les directives `PROXY`, `SOCKS`, et `DIRECT` avec chaînes de fallback
           - Bouton **"Tester Proxy/PAC"** envoie une requête HEAD via le proxy résolu et rapporte la latence
           - Dernier résultat et horodatage persistés entre les redémarrages
           - Appliqué à Google Time Sync (HTTP) et Inspecteur de Certificats HTTPS (tunnel SSL CONNECT)
           - Toutes les erreurs retombent silencieusement sur DIRECT — les problèmes proxy ne bloquent jamais l'utilisation normale
      </div>

## Actions en Lots
<div style={{ display: 'flex', gap: '24px', alignItems: 'flex-start', marginBottom: '32px' }}>
      <div style={{ flex: 1 }}>
         <Img src="/images/6_screencap_bulkactions.png" alt="DIG" style={{ maxWidth: '100%', width: '100%', height: 'auto', border: '1px solid #e5e7eb', borderRadius: '8px' }} />
      </div>
      <div style={{ width: '50%' }}>
         - Charger un fichier de configuration JSON définissant plusieurs commandes diagnostiques à exécuter en séquence
         - Supporte les types de commandes intégrés : `ping`, `dig`, `ntp`, `port-scan`, `checkcert`, `device-info`, `tracert`, `google-timesync`, `lan-scan`
         - Les préfixes de commande inconnus retombent sur l'exécution shell brute
         - Chaque commande s'exécute avec un délai de 30 secondes ; les erreurs et délais d'attente sont capturés par commande sans arrêter le lot
         - Barre de progression en temps réel avec statut par commande (SUCCÈS / ERREUR / DÉLAI D'ATTENTE)
         - Sortie style terminal avec résultats codés par couleur
         - Sauvegarde automatique des résultats dans le chemin de `fichier-de-sortie` défini dans la config après exécution (avec validation d'accessibilité en écriture)
         - Bouton **"Valider Config"** vérifie la structure JSON et l'accessibilité en écriture du chemin de sortie, suggère un chemin de fallback si besoin
         - Bouton **"Écrire dans le fichier"** lance un sélecteur SAF pour sélectionner manuellement l'emplacement de sauvegarde
         - Gestion gracieuse des erreurs : JSON invalide, chemins inaccessibles en écriture, échecs réseau — tous capturés et affichés sans planter
      </div>
</div>


## Traceroute

<div style={{ display: 'flex', gap: '24px', alignItems: 'flex-start', marginBottom: '32px' }}>
      <div style={{ flex: 1 }}>
         - Saisissez n'importe quel nom d'hôte ou adresse IP
         - Appuyez sur **Traceroute** pour démarrer — le bouton devient rouge et indique **Arrêter** pendant l'exécution
         - La sortie s'affiche saut par saut dans une carte terminal monospace avec défilement
         - Implémenté via `ping -c 1 -t <TTL>` pour sonder (aucun binaire `traceroute` requis)
         - Chaque saut qui répond avec Time Exceeded révèle son IP et son temps aller-retour
         - Sonde jusqu'à 30 sauts ; s'arrête automatiquement quand la destination répond
         - L'icône d'historique reflète le résultat :
           - ✅ Destination atteinte (tous ou la plupart des sauts ont répondu)
           - 🤷‍♂️ Certains sauts ont répondu mais la destination n'a pas été atteinte
           - ❌ Aucun saut n'a répondu
         - 5 derniers hôtes tracés conservés en historique cliquable (persisté entre les redémarrages)

## Google Time Sync
<div style={{ display: 'flex', gap: '24px', alignItems: 'flex-start', marginBottom: '32px' }}>
      <div style={{ flex: 1 }}>
         <Img src="/images/10_screencap_ggtimesync.png" alt="DIG" style={{ maxWidth: '100%', width: '100%', height: 'auto', border: '1px solid #e5e7eb', borderRadius: '8px' }} />
      </div>
      <div style={{ width: '50%' }}>
         - Interroge `http://clients2.google.com/time/1/current` via HTTP et analyse la réponse de temps de Google
         - Supprime le préfixe de protection XSSI `)]}'` avant l'analyse JSON
         - Calcule les métriques complètes de synchronisation de type NTP :
           - 📡 **RTT** — temps aller-retour (T4 − T1)
           - ⏱ **Décalage d'horloge** — `correctedServerTime − T4` (positif = horloge locale en retard, négatif = en avance)
           - 🧮 **Heure du serveur corrigée** — `serverTime + RTT / 2`
         - Indicateur de décalage codé par couleur : 🟢 < 100 ms · 🟠 < 500 ms · 🔴 ≥ 500 ms
         - Champ hôte personnalisé (défaut : `clients2.google.com`)
         - Bouton **Copier le décalage** en un clic pour les workflows d'ajustement manuel d'horloge
         - États Inactif / Chargement / Succès / Erreur survivant aux rotations et changements de configuration
         - Supporte le routage proxy optionnel quand Proxy PAC est activé dans Paramètres
      </div>
</div>

## Certificat HTTPS
<div style={{ display: 'flex', gap: '24px', alignItems: 'flex-start', marginBottom: '32px' }}>
      <div style={{ flex: 1 }}>
         - Saisissez n'importe quel nom d'hôte et port (défaut : `google.com:443`)
         - Effectue un véritable échange de certificats TLS et extrait la chaîne de certificats de l'homologue
         - Affiche :
            - Noms distingués Sujet / Émetteur (CN, O, OU, C) pour chaque certificat de la chaîne
            - Période de validité (Non avant / Non après) avec jours avant expiration
            - Statut de validité : ✅ Valide · ⚠️ Expiration Proche · ❌ Expiré · ⚠️ Non Approuvé
            - Numéro de série, empreintes SHA-256 et SHA-1 (cliquer pour copier)
            - Noms Alternatifs de Sujet (SANs)
            - Algorithme de clé, taille de clé, algorithme de signature, profondeur de chaîne
         - Pour les certificats **non approuvés** ou **expirés**, affiche la **chaîne complète** — feuille → intermédiaire(s) → racine — avec marqueurs `[Feuille]`, `[Intermédiaire N]`, et `[Racine]` pour inspecter les détails de chaque certificat même quand la validation de confiance échoue
         - 5 derniers hôtes inspectés conservés en historique cliquable (persisté entre les redémarrages)
         - Supporte le tunneling SSL via HTTP CONNECT quand Proxy PAC est activé
      </div>
